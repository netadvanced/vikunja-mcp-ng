# Vikunja API Coverage & Correctness Audit

Audited against the live Vikunja OpenAPI spec (`GET /api/v1/docs.json` on try.vikunja.io, spec version `v2.3.0-1019-g95b7e673`), the `node-vikunja` client library (v0.4.0) this project wraps, and this repo's `src/tools/` at commit `29f4f47` (branch `main`).

Generated by an 11-agent audit, one per API domain, each reading the spec, the node-vikunja source, and the corresponding MCP tool code directly (not from memory).

## Summary

Vikunja documents **169** distinct API operations (method+path). Of those:

| Status | Count | % |
|---|---|---|
| ✅ Implemented | 49 | 29% |
| ⚠️ Implemented (bug) | 14 | 8% |
| 🟡 Partial | 3 | 2% |
| ❌ Not implemented | 103 | 61% |
| **Total** | **169** | 100% |

> Updated 2026-07-17 (Wave D, `waveD-notifications-subscriptions`): the 7
> `/notifications`, `/subscriptions/{entity}/{entityID}`, and
> `/{kind}/{id}/reactions*` rows below moved from ❌ to ✅ — everything else
> in this document remains the original audit snapshot described above.

- **✅ Implemented** — implementation matches the documented endpoint (method, path, request/response fields all checked).
- **⚠️ Implemented (bug)** — the tool exists and calls something, but the audit found a concrete divergence from the documented contract (wrong field name, dropped required param, wrong response shape assumption, etc.). See the Issues table below.
- **🟡 Partial** — works for the common case but silently drops documented capability (e.g. missing pagination/search params).
- **❌ Not implemented** — no MCP tool calls this endpoint at all. Many of these are intentional (admin, OIDC login, migration importers, TOTP/avatar/CalDAV account-settings minutiae) rather than gaps — see notes per row.
