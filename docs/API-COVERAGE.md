# Vikunja API Coverage & Correctness Audit

Audited against the live Vikunja OpenAPI spec (`GET /api/v1/docs.json` on try.vikunja.io, spec version `v2.3.0-1019-g95b7e673`), the `node-vikunja` client library (v0.4.0) this project wraps, and this repo's `src/tools/` at commit `29f4f47` (branch `main`).

Generated by an 11-agent audit, one per API domain, each reading the spec, the node-vikunja source, and the corresponding MCP tool code directly (not from memory).

- **✅ Implemented** — implementation matches the documented endpoint (method, path, request/response fields all checked).

## Composite / MCP-only Features

These MCP tools don't map 1:1 to a single Vikunja endpoint — they compose several documented calls into one higher-level operation. Listed separately since "not implemented" / "implemented" doesn't apply cleanly.

| Feature | Files | Notes |
|---|---|---|
| vikunja_task_bulk (bulk-create), vikunja_tasks (bulk-create) | `src/tools/task-bulk.ts, src/tools/tasks/bulk-operations-simplified.ts` | Composes PUT /projects/{id}/tasks per task + POST /tasks/{id}/labels/bulk (via setTaskLabels workaround) + PUT /tasks/{id}/assignees per assignee. All underlying calls verified correct against the OpenAPI schemas. |
| vikunja_task_bulk (bulk-update), vikunja_tasks (bulk-update) | `src/tools/task-bulk.ts, src/tools/tasks/bulk-operations-simplified.ts, src/tools/tasks/bulk/BulkOperationValidator.ts` | Composes GET+POST /tasks/{id} per task, plus PUT/DELETE /tasks/{id}/assignees or the setTaskLabels workaround for assignees/labels fields.
